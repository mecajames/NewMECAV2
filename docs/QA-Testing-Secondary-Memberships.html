1. Critical Path Tests

These tests must pass for the feature to be considered functional

TC-001 Create Secondary Membership (Admin) Critical

Prerequisites:

Logged in as Admin, have a user with PAID membership

Test Steps:
  1. Navigate to /admin/members
  2. Click "Add User" button
  3. Select "Membership User"
  4. Enter user details (email, name, password)
  5. On "Secondary Option" step, select "Yes - this is a secondary membership"
  6. Search for a master account by email or MECA ID
  7. Select the master from search results
  8. Choose "No" for give own login
  9. Complete membership type selection
  10. Enter competitor name and vehicle info
  11. Select payment method (e.g., "Cash")
  12. Click "Create Secondary Membership"
Secondary membership created with PENDING status. Master membership shows account type as MASTER. Secondary appears in master's secondary list.

TC-002 MECA ID Assignment on Payment Critical

Prerequisites:

Secondary membership exists with PENDING status (from TC-001)

Test Steps:
  1. Navigate to the secondary membership detail page
  2. Click "Mark as Paid" or process payment
  3. Enter payment amount and transaction ID (if applicable)
  4. Confirm payment
Secondary membership status changes to PAID. A unique MECA ID is assigned to the secondary. MECA ID is different from the master's MECA ID.

TC-003 Secondary with Own Login Critical

Prerequisites:

Logged in as Admin, have a master account, have unused email address

Test Steps:
  1. Create a secondary membership following TC-001 steps
  2. At "Give own login" step, select "Yes"
  3. Enter a unique email address for the secondary
  4. Complete membership creation
  5. Log out of admin account
  6. Log in with the secondary's email (password may need reset)
Secondary can log in with their own credentials. Dashboard shows NO billing section/tab. Secondary can view their profile and competition results. Secondary cannot access invoices or payment methods.

TC-004 MECA ID Switcher for Master Critical

Prerequisites:

Master account with at least 2 paid secondary memberships

Test Steps:
  1. Log in as the master account holder
  2. Navigate to My MECA dashboard
  3. Locate the MECA ID switcher dropdown in the header
  4. Click the dropdown
  5. Verify all MECA IDs are listed (master's own + secondaries)
  6. Select a secondary's MECA ID
Dashboard shows data for the selected MECA ID. Dropdown shows current selection. Master can see competition results for the selected secondary.

TC-005 Upgrade Secondary to Independent Critical

Prerequisites:

Secondary membership with PAID status

Test Steps:
  1. Log in as Admin
  2. Navigate to the secondary's member detail page
  3. Click "Upgrade to Independent" action
  4. Confirm the upgrade
  5. Verify the membership details
Account type changes from SECONDARY to INDEPENDENT. The same MECA ID is preserved. Master relationship is removed. If secondary had own login, their billing access is now enabled.

2. Admin Workflow Tests

TC-006 Master Search Functionality High

Test Steps:
  1. Start creating a secondary membership (Admin wizard)
  2. At secondary option step, select "Yes"
  3. In search field, enter master's email
  4. Verify results appear
  5. Clear and search by MECA ID
  6. Verify same master appears
  7. Clear and search by partial name
Search works by email, MECA ID, and name. Results show MECA ID, name, and email. Only PAID memberships appear in results. SECONDARY memberships do NOT appear (can't be master).

TC-007 View Secondary List on Master Detail High

Test Steps:
  1. Navigate to Admin → Members
  2. Click on a user who has master status
  3. Scroll to "Secondary Memberships" section
  4. Verify all secondaries are listed
Table shows: MECA ID, Name, Type, Has Login, Status, Actions. Each secondary has View, Remove, and Upgrade buttons. Counter shows "X of 10 secondaries".

TC-008 Remove Secondary from Master High

Test Steps:
  1. Navigate to master's member detail page
  2. In Secondary Memberships section, click "Remove" on a secondary
  3. Confirm the removal
  4. Navigate to the former secondary's detail page
Secondary removed from list. Former secondary now shows INDEPENDENT account type. MECA ID is preserved. No data loss occurred.

TC-009 Add Secondary Button Visibility Medium

Test Steps:
  1. Navigate to a user with PENDING membership
  2. Look for "Add Secondary" button
  3. Navigate to a user with PAID membership
  4. Look for "Add Secondary" button
"Add Secondary" button is NOT visible for PENDING memberships. "Add Secondary" button IS visible for PAID memberships.

3. User Dashboard Tests

TC-010 User Adds Secondary from Dashboard High

Prerequisites:

Logged in as user with PAID independent membership

Test Steps:
  1. Navigate to My MECA dashboard
  2. In "My Memberships" section, click "Add Secondary"
  3. Complete the Add Secondary modal wizard
  4. Submit and pay for the secondary
User's account type changes to MASTER. New secondary appears in their secondaries list. MECA ID switcher appears in header.

TC-011 Secondary Cannot See Billing High

Prerequisites:

Secondary account with own login

Test Steps:
  1. Log in as secondary account
  2. Navigate through all dashboard tabs
  3. Try to access /billing directly via URL
  4. Look for any billing/payment related UI
No billing tab visible in navigation. Direct URL access shows message "Billing managed by [Master Name]". No payment buttons or invoice links visible.

TC-012 Secondary Can Edit Own Profile Medium

Test Steps:
  1. Log in as secondary with own login
  2. Navigate to Profile section
  3. Edit profile details (name, phone, etc.)
  4. Edit vehicle information
  5. Save changes
Secondary can edit their own profile. Changes are saved successfully. Vehicle information updates correctly.

4. Billing & Payment Tests

TC-013 Secondary Invoice Goes to Master High

Test Steps:
  1. Create a secondary with Credit Card payment method
  2. Check where the invoice was sent
  3. View the invoice details
Invoice email sent to master's email address. Invoice shows master as billing contact. Invoice line item shows secondary's name and MECA ID.

TC-014 Independent Renewal Dates Medium

Test Steps:
  1. Create two secondaries on different dates
  2. View each secondary's membership details
  3. Check start and end dates
Each secondary has their own independent start/end dates. Renewal dates are not tied to master's renewal. Master would receive separate renewal invoices.

TC-015 Complimentary Secondary Low

Test Steps:
  1. Create secondary with "Complimentary" payment method
  2. Enter reason for complimentary membership
  3. Complete creation
Secondary created with PAID status ($0). MECA ID assigned immediately. No invoice generated.

5. Edge Cases & Error Handling

TC-016 Max Secondaries Limit Medium

Test Steps:
  1. Create a master with 10 secondaries
  2. Try to add an 11th secondary
Error message displayed: "Maximum of 10 secondary memberships allowed". Add Secondary button should be disabled or show limit reached.

TC-017 Duplicate Email Prevention Low

Test Steps:
  1. Create secondary with own login using email "test@example.com"
  2. Try to create another secondary with same email
Error message: "A profile with this email already exists".

TC-018 Cannot Add Secondary to Secondary Low

Test Steps:
  1. Search for a SECONDARY membership in master search
  2. Try to select it as master
Secondary memberships do not appear in master search results. API returns error if attempted directly.

6. Regression Checklist

Existing Functionality

  • Standard user creation still works (non-secondary)
  • Standard membership purchase still works
  • Invoice generation works for regular memberships
  • MECA ID assignment works for regular memberships
  • Event registration still works
  • Competition results still display correctly
  • Member search/listing works correctly
  • Profile editing works for regular users

API Endpoints

  • POST /memberships/:id/upgrade-to-master - Returns 200
  • POST /memberships/:id/secondaries - Creates secondary correctly
  • GET /memberships/:id/secondaries - Returns list
  • GET /memberships/:id/master-info - Returns master info with secondaries
  • DELETE /memberships/:id/secondaries/:secondaryId - Removes correctly
  • POST /memberships/:id/upgrade-to-independent - Upgrades correctly
  • GET /memberships/user/:userId/controlled-meca-ids - Returns all IDs
