Business Logic: Communication

How the platform talks to members and admins: in-app notifications, transactional emails, support tickets, and external newsletter sync.

In-app Notifications

Overview
Bell-icon notifications shown in the member nav. Sent by admins and by the platform itself.
Technical
notifications per-recipient row with title, body (markdown), type, link, read flag, fromUserId. Service exposes per-recipient and broadcast (all-active / all-users / specific list) creation.
Why
In-app delivery is read-anywhere, doesn't hit spam folders, and is free to send. Per-row storage (vs single-row-with-recipient-list) lets each user mark their own copy as read independently.
Manage
/admin/notifications + per-row Send Message on /admin/members. Notifications guide.

Transactional Email System

Overview
34 templates across account, membership, payment, event, shop, ticket, and judge/ED categories.
Technical
Service-per-template (sendMembershipWelcomeEmail, sendRefundConfirmationEmail, etc.). Provider-agnostic: SendGrid, Resend, Mailgun, or Nodemailer SMTP based on Site Settings. Each template renders HTML + plain-text fallback.
Why
Transactional events deserve email (members don't live in our app). Provider-agnostic delivery lets us swap providers without changing every send-call site.
Manage
/admin/email-testing for previews. Email Reference Matrix.

Admin Alert Emails

Overview
Automatic emails to admin staff for revenue events, dispute alerts, and weekly digests.
Technical
AdminNotificationsService fires emails on new memberships, new shop orders, payment failures, dispute creation, dispute losses, refunds, and the Sunday weekly digest.
Why
Owners want to know when revenue lands and when something needs attention. Automating the alerts means nothing slips through if no one is watching the dashboard.
Manage
Recipient list configured in Site Settings.

Support Tickets (member & guest)

Overview
Member tickets and guest (non-logged-in) tickets via the support desk.
Technical
tickets with subject, category, body, status. Members file directly. Guests file via /support/guest, then verify email before getting an access token to view/reply. Admin replies and status changes both fire emails.
Why
Guest tickets exist because some support requests come from people who can't log in (account locked, didn't finish signup). Email verification prevents support-spam abuse.
Manage
/admin/tickets. Support Tickets guide.

Constant Contact Sync

Overview
New members are pushed to a Constant Contact list for newsletter delivery.
Technical
New profile creation queues a Constant Contact API push. Sync runs on a schedule.
Why
Constant Contact is where MECA's marketing newsletter lives. Auto-sync new members so they're in the audience without admin manual entry.
Manage
Connection status on /admin/site-settings → Integrations.
