Apply a Manual Cash / Check Payment

Two paths: early renewal via Add Membership wizard, or marking an existing pending membership PAID

/admin/members → member detail → Memberships tab

Two real flows depending on what already exists

If the member doesn't have a pending membership yet (typical event-day scenario — they pay you cash for an upcoming renewal while their current one is still active), use Path A — the Add Membership wizard. If a pending membership already exists on the account (e.g. created earlier by a renewal flow that was never paid), use Path B — the Apply Payment button on that pending row.

2. Path A: Add Membership wizard (most common)

1
Open the member's detail page
From /admin/members click their row.
2
Open the Memberships tab → Assign Membership
Click the orange "Assign Membership" button at the top right of the Memberships section. The wizard launches.
3
Pick the membership type and fill required fields
Same flow as any other admin-created membership — type, vehicle info (for competitors), business info (for retailers/manufacturers), team add-on if applicable.
4
On the Payment Method step, pick Cash or Check
A blue info banner reminds you that early renewal is handled automatically. For Check, enter the check number (required). For Cash, optionally enter a receipt number.
5
Save
The system detects whether the member already has an active membership of the same category. If yes, the new term starts the day after the current one expires (no overlap). If no, the new term starts today.
6
Verify
Two memberships now appear on the Memberships tab — the still-active current one (unchanged) and a new "future-dated" one that will activate when the current expires. The member's billing history shows a new Order + paid Invoice for the cash/check amount.

3. Path B: Apply Payment button on a pending membership

If a pending membership already exists on the account (the row shows the yellow "pending" badge), use this path:

1
Open the member's Memberships tab
Find the pending membership row.
2
Click the green "Apply Payment" button
Visible only on rows where paymentStatus = pending and the membership hasn't expired.
3
Fill the modal
Pick Cash or Check. For Check, enter check number (required). For Cash, receipt number is optional. Amount defaults to the membership type price; override if the member paid a different amount. Notes are free-text — typical use is "paid at March 15 event to ED Smith."
4
Save
Membership flips to PAID. An Order and Invoice are generated and marked PAID. The member's profile expiration updates.

4. The auto-extend math

For Path A only — Path B leaves the existing pending membership's start/end dates as-is and just marks them paid.

The rule

If member has an existing ACTIVE PAID membership of the same category (cancelledAt null, endDate in the future):

  • New start = day after current end date
  • New end = new start + 365 days
  • Effect: member's expiration moves out by exactly one year, with no overlap.

If member has no active membership (or only an expired one): new start = today, new end = today + 365.

Example: today is March 15. Member's current membership expires April 15 (30 days from now). They hand you $50 cash for renewal.

  • You go through the Add Membership wizard, pick Cash, enter receipt #1234, save.
  • System sees an active membership ending April 15. New term starts April 16, ends April 15 next year.
  • Member keeps full access through April 15, then auto-rolls into the new term on April 16. No gap, no double-billed days.
  • An Order and a paid Invoice are created for $50, transactionId = CASH-1234.

6. Audit trail

Open /admin/audit-log and filter by action:

  • membership_admin_create — Path A entries.
  • membership_apply_manual_payment — Path B entries.

Each entry shows the admin who recorded the payment, the member, and the amount + reference. QuickBooks sync (if connected) mirrors the new Invoice within the next sync window.
