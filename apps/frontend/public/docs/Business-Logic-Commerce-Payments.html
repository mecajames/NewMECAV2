Business Logic: Commerce & Payments

How money moves through the system: Stripe, PayPal, manual cash entries, refunds, disputes, the shop, and accounting sync.

Stripe Payment Processing

Overview
Primary payment provider for memberships, event registrations, and shop purchases.
Technical
Stripe Elements tokenizes the card client-side. Backend creates a PaymentIntent (one-time) or Subscription (recurring). Webhooks fire on payment_succeeded, invoice.payment_failed, customer.subscription.updated, charge.dispute.created. Each webhook updates membership / order state and triggers the right email.
Why
Webhooks are the source of truth for "did the charge actually go through?" — the synchronous create call returns "pending" half the time. Listening to webhooks instead of polling means real-time reconciliation.
Manage
Webhook URL configured in Site Settings + Stripe dashboard. Disputes surfaced via admin alert emails.

PayPal Checkout

Overview
Alternative checkout path for members who prefer PayPal.
Technical
PayPal SDK on the frontend; backend captures the order via PayPal API on approval. Order maps to the same membership/registration/shop record as a Stripe purchase.
Why
PayPal carries an older audience that doesn't want to type a card number. Supporting it widens the converting funnel.
Manage
PayPal credentials in Site Settings → Integrations.

Coupons / Discount Codes

Overview
Promotional codes that apply at checkout to reduce membership or shop totals.
Technical
coupons with code, type (percent / fixed), value, scope (memberships / shop / both), expiration, usage cap. Checkout validates redemption against scope, expiration, and remaining caps before applying.
Why
Scoping codes (memberships only / shop only) prevents accidental cross-product discounts. Per-coupon usage cap supports "first 50 redemptions" promos without engineering work.
Manage
/admin/coupons. Coupons guide.

Manual Cash / Check Payments

Overview
Admin records offline payment against a membership or invoice.
Technical
Admin form on member detail captures method (cash / check), amount, date, reference (check number), optional invoice link. Inserts an order with method=manual; if linked to a pending invoice, marks it paid.
Why
Real-world events get cash and checks. The platform must mirror these so QuickBooks reflects total revenue and the member sees their membership as paid.
Manage
Member detail. Manual Cash Payment guide.

Refunds (full, partial, dispute)

Overview
Three refund triggers: admin-initiated full / partial, and Stripe-initiated (dispute lost).
Technical
Admin full refund: Stripe createRefund full amount, membership cancelled, refund email. Partial: Stripe createRefund partial, membership stays active, note appended. Dispute lost: Stripe webhook fires, we mirror the chargeback as a refund-equivalent and notify admins.
Why
Three triggers, one downstream pipeline (mark refunded → notify member → audit log). The dispute path is automatic because we don't want to forget to recognize a chargeback we already lost.
Manage
Member detail or bulk Refund all. Membership Workflows deep guide.

Shop Products & Orders

Overview
Storefront with inventory tracking, fulfillment workflow, and admin order management.
Technical
shop_products with name, SKU, price, image, stock count, is_active. shop_orders with status (pending / shipped / delivered / refunded), tracking number, shipping address. Stock decrements on order; goes back on refund.
Why
Real inventory tracking (vs unlimited) prevents overselling popular items. Status field drives the customer-facing order detail page and the admin fulfillment queue.
Manage
/admin/shop/products and /admin/shop/orders. Shop Products + Shop Orders.

QuickBooks Sync

Overview
Mirror invoices and payments into QuickBooks Online so accounting stays in sync.
Technical
OAuth connection to QuickBooks. Service queues new invoices/payments and pushes them on a schedule. Failures retry; persistent failures surface as admin alerts.
Why
Owners want one accounting source of truth. Auto-sync prevents the typical "manually export CSV every Monday" toil.
Manage
/admin/quickbooks. QuickBooks Sync guide.

Invoices

Overview
Formal invoice records used for QuickBooks reconciliation and disputed-charge documentation.
Technical
invoices generated automatically for memberships, registrations, shop orders. Manual invoices created from admin for off-platform billing. PDF generated on demand.
Why
Receipts (in-app) ≠ invoices (formal documents). Some retailers and corporate sponsors require a numbered invoice for their AP. Auto-generation means we never miss one.
Manage
/admin/billing/invoices.
