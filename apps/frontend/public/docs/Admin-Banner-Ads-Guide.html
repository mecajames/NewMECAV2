Banner Ad System — Sizes, Placements, Creative Specs

9 IAB sizes × 15 placements, frequency capping, rotation weight, and a creative brief for advertisers

/admin/banners · /admin/advertisers · /admin/banners/analytics

Banner inventory

The platform serves banner ads in 9 standard IAB sizes across 15 placements. Each banner targets a specific advertiser, runs in a date window, and rotates against other active banners on the same placement. Engagement (impressions and clicks) is tracked per banner and per user — the platform supports frequency capping (max impressions per user per banner) and rotation weighting.

Creative requirements

  • File formats — PNG, JPG, or animated GIF. SVG also accepted for logos.
  • Max file size — 200 KB per image. Optimize aggressively; the platform serves millions of impressions and bandwidth matters.
  • Resolution — supply at 2× the listed dimensions (e.g. 1456×180 for a Leaderboard) so the banner stays sharp on retina screens.
  • Animation — animated GIFs allowed but max 3 loops, max 15 seconds total.
  • Color profile — sRGB only.
  • Click URL — must be HTTPS. The system tracks clicks regardless but rejects insecure URLs at upload.
  • Alt text — required for accessibility; descriptive of the ad content.

3. Create and schedule a banner

1
Make sure the advertiser exists
Open /admin/advertisers. Add the advertiser if they're new — company name, contact name/email/phone, optional website and notes.
2
Create the banner
Open /admin/banners → "+ New Banner". Pick the advertiser. Set internal name (for admin reference). Upload the image — pick the size that matches the creative.
3
Pick placements
Tick one or more positions (a banner can run on multiple placements simultaneously if the size fits). The recommended placements for the chosen size are highlighted.
4
Set click URL and alt text
HTTPS required. Alt text describes the ad to screen readers and is also used for SEO.
5
Set the schedule
Start date and end date. Outside this window the banner is automatically hidden — no need to remember to take it down.
6
Set status
DRAFT (still being prepared, not shown), ACTIVE (live within date window), PAUSED (admin-suspended even within window), ARCHIVED (retired).
7
Save
Banner goes live within seconds if status = ACTIVE and current date is within the window.

4. Frequency capping & rotation weight

SettingDefaultWhat it does
maxImpressionsPerUser0 (unlimited)Stop showing this banner to a user after N impressions. Use to prevent annoying repeated exposure.
maxTotalImpressions0 (unlimited)Hard cap on total impressions across all users. Use for fixed-impression campaigns ("buy 100,000 impressions").
rotationWeight100Higher value = more likely to be shown when multiple banners qualify for the same slot. Set 200 to roughly double rotation share vs a default-weight competitor.
priority0Hard tiebreak — higher priority always shown ahead of lower priority within the same placement. Use sparingly; rotation weight is usually preferred.

5. Sample creative requirements brief for advertisers

When pitching to a new advertiser, send them this brief:

MECA banner ad creative requirements

Sizes available (any IAB standard listed above). Most popular: 728×90 Leaderboard, 300×250 Medium Rectangle, 970×250 Billboard.

Format: PNG / JPG / animated GIF / SVG. Max 200KB. Supply at 2× resolution for retina.

Animation: optional, max 15 seconds, max 3 loops.

Click URL: must be HTTPS.

Alt text: ~80 chars describing the ad for accessibility.

Lead time: 2 business days from creative receipt to campaign launch.

6. Banner lifecycle

  • DRAFT — admin is preparing; not shown publicly.
  • ACTIVE — within date window AND status = ACTIVE; eligible for rotation.
  • PAUSED — admin temporarily hid the banner without changing the schedule. Useful for disputes or content review.
  • ARCHIVED — campaign finished. Kept in the system for analytics but not in active rotation.

7. Analytics — what's tracked

Every impression and click is logged in banner_engagements with banner id, user id (if logged in), session id, page URL, and timestamp. The Banner Analytics page (/admin/banners/analytics) aggregates this into:

  • Per-banner totals — impressions and clicks for the selected date range.
  • CTR — clicks ÷ impressions, color-coded against industry benchmarks.
  • Daily breakdown — chart per banner showing trends.
  • Top placements — which positions generate the most clicks for this banner.
  • CSV export — for monthly advertiser reports. Columns: date, banner, impressions, clicks, CTR.

Click events also fire a server-side redirect through /banners/<id>/click so click attribution is reliable even if the destination loses query params.
