Manage Members

Search, create, edit, impersonate, and delete member accounts

/admin/members

Audience: site administrators

The Members page is the operational hub for every account on the platform. Use it to find a member, view their full history, edit their record, or perform privileged actions like impersonation.

1. Find a member

1
Type into the search box
filters across name, email, and MECA ID instantly.
2
Use the Role / Membership Type / Status filters to narrow further.
3
Tick "Show online only" to see who's currently logged in.
4
Click any row to open the member detail page (anywhere except the checkboxes/buttons).

2. Create a new member

1
Click "+ New Member" at the top right.
2
Choose account type
Member for full members, Basic for converted guest registrations.
3
Fill name, email, phone, password approach (auto-generate / set manually / send-reset-link), force password change flag.
4
Optionally assign a membership type and pick a payment method (test purchase, manual entry, or comp).
5
Save
the new account appears at the top of the members list with their MECA ID assigned automatically.

3. Edit a member's record

1
Open the member detail page.
2
Edit profile fields
name, email, phone, profile picture, public visibility flag, vehicle info.
3
Edit addresses
primary, billing, shipping. Tick "Use billing for shipping" to mirror.
4
Edit permissions
staff access, judge / event director permissions and certification expiration dates.
5
Save each section
every staff/role change is captured in the admin audit log.

4. Impersonate a member

1
Open the member detail page.
2
Click Impersonate
generates a one-time magic link.
3
Open that link in a new browser tab
you're signed in as the member and can see exactly what they see.
4
Log out of the impersonation when done
your admin session is unaffected.

Note

Use impersonation for support — for example, to confirm a user's reported bug or walk them through a feature. Impersonation is logged in the audit trail.

5. Send a notification

1
Click "Send Message" on any member row.
2
Type a subject and body
Markdown is supported.
3
Send
the message lands in the recipient's notification bell within seconds.

6. Delete a test member

1
Open the member detail page.
2
Use the Delete option
confirm the warning.
3
The user is removed from both the profiles table and the auth system.

Caution

Only delete test accounts. For real accounts with payment or registration history, deactivate or ban instead — deletion cascades and removes historical references.
