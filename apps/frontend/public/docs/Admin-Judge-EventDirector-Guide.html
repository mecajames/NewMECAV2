1. System Overview

The MECA Judge and Event Director system provides two ways to onboard judges and event directors:

  1. Application Process - Members apply through the website, admin reviews and approves
  2. Direct Creation - Admin creates judge/ED records directly without an application

Both methods work with a permission system that controls what members see in their dashboard and whether they can be assigned to events.

2. Key Concepts

Judge/ED Record

A database record that stores information about the judge or event director, including:

  • Level (for judges): In Training, Apprentice, Standard, Senior, Head
  • Specialty: SQL, SPL, or Both
  • Location information: City, State, Country
  • Statistics: Events judged/directed, ratings
  • Active Status: Whether they're currently active in the system

Permission Toggle

A setting on the member's profile that controls:

  • Whether they see judge/ED sections in their dashboard
  • Whether they can be assigned to events
/admin/members/{id} → Permissions Tab

Important: Active Status vs Permission

These are two separate concepts:

  • Active Status (on Judge/ED record) - Whether the judge/ED is active in the system
  • Permission Toggle (on Member profile) - Whether they see dashboard buttons and can be assigned to events

Both must be enabled for a judge/ED to be assigned to events.

3. Application Process

The traditional workflow where members apply to become judges or event directors.

1
Enable Permission
Admin goes to member's profile → Permissions tab → Enable "Judge" or "Event Director" permission
2
Member Sees Application Link
Member now sees "Become a Judge" or "Become an Event Director" in their dashboard
3
Member Submits Application
Member fills out the full application form with experience, references, essays, etc.
4
Admin Reviews Application
Admin reviews at:
  • /admin/judge-applications for judges
  • /admin/event-director-applications for event directors
5
Admin Approves
Upon approval, a Judge/ED record is created and member sees "My Judging" or "My Event Directing" in dashboard

4. Direct Creation

For situations where you want to add a judge or event director without going through the application process.

When to Use Direct Creation

  • Importing existing judges from a previous system
  • Adding established industry professionals who don't need to apply
  • Creating judge/ED records for testing purposes
  • Backdating records for members who were judges before the system existed

How to Create Directly

1
Navigate to Admin Page
Go to /admin/judges or /admin/event-directors
2
Click "Create Judge" or "Create Event Director"
Button is in the top right corner of the page
3
Search and Select Member
Search by name, email, or MECA ID. Their city/state will auto-populate from their profile if available.
4
Fill in Details
  • For Judges: Select level and specialty
  • State and City (required)
  • Admin notes (optional)
5
Choose Permission Setting
Important Check or uncheck "Enable dashboard permission":
  • Checked: Member immediately sees dashboard buttons and can be assigned to events
  • Unchecked: Record is created but member won't see dashboard buttons until you enable permission separately
6
Click Create
The judge/ED record is created with Active status

5. Application vs Direct Creation

Application Process

  • Member initiates the process
  • Full application with essays, references
  • Admin review workflow
  • Application record stored for audit
  • Good for vetting new judges/EDs
  • Permission must be enabled first for member to see apply link

Direct Creation

  • Admin initiates the process
  • Minimal information required
  • Instant creation, no review needed
  • No application record (created directly)
  • Good for imports and established judges
  • Permission can be enabled at creation time or later

6. Understanding Permissions

Permission Toggle Location

/admin/members/{member-id} → Click "Permissions" tab

What Permission Controls

Permission When ON When OFF
Judge Permission
  • Member sees Judge section in dashboard
  • Can be assigned to events as judge
  • Shows "My Judging" or "Become a Judge" link
  • Judge section hidden from dashboard
  • Cannot be assigned to events
  • Judge record still exists (if created)
Event Director Permission
  • Member sees ED section in dashboard
  • Can be assigned to events as ED
  • Shows "My Event Directing" or "Become an Event Director" link
  • ED section hidden from dashboard
  • Cannot be assigned to events
  • ED record still exists (if created)

7. Status Matrix

This table shows all possible combinations and what they mean:

Record Exists Active Status Permission Dashboard Shows Can Assign
No N/A ON "Become a Judge/ED" link No
Yes Active ON "My Judging/ED" link Yes
Yes Active OFF Nothing (hidden) No
Yes Inactive ON Nothing (hidden) No
Yes Inactive OFF Nothing (hidden) No

Key Takeaway

To see the "My Judging/ED" link in dashboard AND be assignable to events, ALL of these must be true:

  1. Judge/ED record must exist
  2. Active status must be ON (on the judge/ED record)
  3. Permission must be ON (on the member's profile)

If any condition is false, the Judge/ED section is hidden from the dashboard.

8. Common Scenarios

Scenario 1: New Member Wants to Be a Judge

  1. Member contacts admin expressing interest
  2. Admin goes to member's profile → Permissions → Enable Judge permission
  3. Member sees "Become a Judge" in dashboard and submits application
  4. Admin reviews and approves at /admin/judge-applications
  5. Member now sees "My Judging" in dashboard

Scenario 2: Import Existing Judge from Old System

  1. Admin goes to /admin/judges → Click "Create Judge"
  2. Search for member and select them
  3. Set appropriate level and specialty
  4. Check "Enable dashboard permission"
  5. Click Create - member immediately has full judge access

Scenario 3: Create Judge Record but Don't Grant Access Yet

  1. Admin goes to /admin/judges → Click "Create Judge"
  2. Search for member and select them
  3. Set level and specialty
  4. Leave "Enable dashboard permission" unchecked
  5. Click Create - judge record exists but member sees nothing
  6. Later, go to member's profile → Permissions → Enable Judge to grant access

Scenario 4: Temporarily Disable a Judge

Option A: Set judge to Inactive at /admin/judges/{id}

  • Judge section hidden from member's dashboard (requires both Active AND Permission)
  • Judge cannot be assigned to events

Option B: Disable permission on member's profile

  • Judge section hidden from member's dashboard
  • Judge cannot be assigned to events

Either option achieves the same user-facing result (hidden dashboard section, cannot be assigned).

Scenario 5: Judge Resigns

  1. Go to /admin/judges/{id}
  2. Set Status to "Inactive"
  3. Optionally: Go to member's profile → Permissions → Disable Judge permission
  4. The judge record remains for historical purposes (events they judged, etc.)

9. Troubleshooting

Member can't see Judge/ED section in dashboard

Check:

Is the permission enabled on their profile?

/admin/members/{id} → Permissions tab → Ensure Judge or ED toggle is ON

Can't assign judge to an event

Check:

  1. Does the judge record exist? Check at /admin/judges
  2. Is the judge Active? Check at /admin/judges/{id}
  3. Is the permission enabled? Check at /admin/members/{id} → Permissions

All three must be true to assign a judge to an event.

Member sees "Become a Judge" but already has a Judge record

This shouldn't happen normally. If it does:

  1. Check if there are duplicate records
  2. Verify the judge record is linked to the correct user_id
  3. Try refreshing the page or clearing browser cache

Application was approved but member doesn't have access

Check:

  1. Was the application actually approved? Check status at /admin/judge-applications/{id}
  2. Did approval create a judge record? Check at /admin/judges
  3. Is permission enabled? When applications are approved, permission should auto-enable, but verify at /admin/members/{id} → Permissions
