Admin Dashboard Overview

Where to find every admin tool, what it does, and how to use it

Who this guide is for

Site owners and operators who manage the MECA platform day-to-day. It covers every section of the admin dashboard except a small set of super-admin-only operations (e.g. modifying MECA IDs) that are restricted to the development team. If a button or page exists in your account, this guide explains it.

1. Dashboard Home

/dashboard/admin

The admin dashboard home is your jumping-off point. Cards on this page link directly to every other admin section described in this guide. Use the search bar at the top to jump to any member by name, email, or MECA ID without navigating manually.

2. Member Management

/admin/members

The Members page lists every profile on the platform with searchable columns for name, email, MECA ID, role, membership type, status, auto-renew, and join date. Each row expands to show secondary (sub-account) members attached to the primary.

2.1 Find & view members

  • Type into the search box to filter by name, email, or MECA ID.
  • Use the Role / Membership Type / Status filters to narrow the list.
  • Tick "Show online only" to focus on currently-active members.
  • Click any row (anywhere except checkboxes/buttons) to open the member detail page.

2.2 Create a new member

  1. Click + New Member at the top right.
  2. Choose account type: Member for full members, Basic for guest registrations being converted to accounts.
  3. Enter first/last name, email, phone, and choose a password approach: auto-generate, set manually, or send a reset link by email.
  4. Optionally tick Force password change on first login.
  5. Optionally pick a membership type and payment method (test purchase, manual entry, comp). The MECA ID is assigned automatically.
  6. Save. The new member appears at the top of the list with their MECA ID and status.

2.3 Edit a member's record

Open the member detail page. Every section is editable in place:

  • Profile — name, email, phone, profile picture, public visibility, vehicle & audio system info.
  • Addresses — primary, billing, and shipping; tick "Use billing for shipping" to mirror them.
  • Permissions — staff access, judge / event director permissions and certification expiration dates.
  • Membership — see Membership Management below.

Changes are saved per-section and immediately persist. Edits to staff/role flags are recorded in the admin audit log.

2.4 Bulk membership actions

Tick the checkbox next to one or more rows to expose the bulk action toolbar:

  • Refund all — full refund + immediate cancellation for every selected membership.
  • Cancel now — immediate cancellation; no refund.
  • Cancel at renewal — keeps the membership active until its current period ends, then prevents auto-renew.

Each bulk action requires a reason of at least 5 characters and shows a per-row success/failure summary so you can see which (if any) failed.

2.5 Apply a manual cash or check payment

  1. Open the member detail page and find the manual payment section.
  2. Pick payment method Cash or Check and enter the amount, plus a reference note (e.g. check number).
  3. Optionally attach the payment to a specific membership or invoice.
  4. Save. The payment appears in the member's billing history with method = cash/check, and in Admin → Billing. If QuickBooks sync is connected, the entry mirrors over within the next sync window.

2.6 Update a member's billing or payment method

  • Edit the billing address (street/city/state/postal code/country) on the member detail page — it'll be used on subsequent invoices.
  • If the member has a saved card on file, the admin tool lets you remove it. The change reflects in their Stripe billing portal.

2.7 Impersonate a member

From the member detail page, click Impersonate. The system generates a one-time magic link that signs you in as that user so you can verify exactly what they see. Log out to return to your admin session.

2.8 Send a notification

Click Send Message on a member's row. Type a subject and body — the message lands in the recipient's notification bell within seconds.

2.9 Delete a test member

Use the Delete option on the detail page only for test accounts with no real activity. Deletion removes the user from both the profiles table and the authentication system and cannot be undone.

3. Membership Management

/admin/memberships · /admin/membership-cards

3.1 Membership Types

The Membership Types page lists every type the system offers (Family, Individual, Team add-on, Forever, etc.). Each type has a configured price, term length, included features, and category (competitor / retail / manufacturer / team).

  • Edit pricing or details inline.
  • Toggle a type inactive to remove it from the public Membership page without deleting the existing memberships using it.

3.2 Assigning a membership to an existing member

From the member detail page:

  1. Click Add Membership.
  2. Pick the type and term.
  3. Choose how the member paid: Stripe (charge the saved card), PayPal, manual cash/check, or comp.
  4. Save. A MECA ID is generated if one isn't already assigned and the membership shows as Active.

3.3 Cancel or refund a membership

From either the member detail page or the bulk toolbar on Members:

  • Cancel at renewal — leaves the current period intact; auto-renew is disabled.
  • Cancel now — ends access immediately.
  • Refund — issues a full Stripe/PayPal refund and ends access immediately.

Every cancellation or refund is captured in the audit log with the admin who initiated it and the reason given.

3.4 Membership Cards

/admin/membership-cards

Tracks the printed/shipped state of physical membership cards. Update statuses as cards move through the workflow.

3.5 Dunning queue

Members whose auto-renew payment failed escalate through dunning steps: day 1, day 3, day 7, and auto-suspend on day 14. The dunning view shows everyone currently in this funnel with their last step and timestamp. The daily cron handles email reminders automatically — admins generally only intervene to resolve disputes or apply manual payments.

4. Event Management

/admin/events

4.1 Create an event

  1. Click + New Event.
  2. Set name, date, venue, address, format(s), and registration window.
  3. Pick an Event Director and Judge panel from the dropdowns (only certified users appear).
  4. Save. The event appears immediately on the public Events page and is opened for registration if the window has begun.

4.2 Edit an event

Open an event from the admin list and edit any field. Changes propagate to the public event page on save. Use this to update a venue change, edit the schedule, or adjust the judge roster.

4.3 Event Registrations & QR Check-In

/admin/event-registrations · /admin/qr-checkin

  • The Registrations page lists every registered competitor with their classes, payment status, and check-in state.
  • The QR Check-In hub lets staff scan a member's digital card (or enter a MECA ID manually) at the event to mark them checked in.
  • Idempotent — scanning the same person twice doesn't double-check them in.

4.4 Enter results

See the dedicated Results Entry System guide for manual entry, Excel upload, and TermLab import. Entered results immediately feed the public Results, Leaderboard, and Standings pages.

4.5 Delete an event

Test events can be deleted entirely. Production events with real results should generally be cancelled (using the cancellation workflow) rather than hard-deleted to preserve history.

5. Judges & Event Directors

/admin/judges · /admin/event-directors · /admin/judge-applications · /admin/event-director-applications · /admin/hosting-requests

For the full lifecycle (application → approval → certification → assignment → expiration), see the dedicated Admin Guide: Judge & Event Director Management. Quick reference:

  • Promote a member to Judge — open their member detail page, grant Judge permission, and approve their submitted application from /admin/judge-applications. They appear in /admin/judges and the public Judges directory.
  • Promote a member to Event Director — same shape, via /admin/event-director-applications.
  • Hosting requests — review submissions from members or venues asking to host a future event, then convert approved requests into actual events.

6. Competition Setup

6.1 Classes

/admin/classes

Define every competition class (e.g. Stock Class 1, Modified, Pro). Each class is bound to a Format and a Season, with optional unlimited-wattage flag and display order. Inactive classes are hidden from registration without losing historical results.

6.2 Formats

/admin/formats

Top-level scoring categories: SPL, SQL, Show & Shine, Dueling Demos, etc. Edit display name and scoring style; changes propagate to public surfaces immediately.

6.3 Seasons

/admin/seasons

Each year is its own season with a start/end date and "current" / "next" flags. The current season drives event eligibility, points calculations, and the Membership renewal cycle.

6.4 Rulebooks

/admin/rulebooks

  • Upload a new PDF rulebook for a specific format/season.
  • Replace an existing rulebook (the public page serves the new version immediately).
  • Archive prior-year rulebooks — they remain accessible from the public Rulebook Archive page.

6.5 Points Configuration

/admin/points-configuration

See Points Configuration Guide for the full point-value matrix. Edit point values per place / class / format here.

7. Billing & Financial

/admin/billing

7.1 Billing Dashboard

Top-level KPIs: revenue this month, year-to-date, recent orders/invoices, refund counts, and the dunning queue size.

7.2 Orders, Invoices, Revenue

  • Orders — every transaction across memberships, shop, and event registrations. Click into any to see line items, payment method, and refund history.
  • Invoices — formal invoice records for QuickBooks reconciliation.
  • Revenue Reports — charts and tables grouped by month, format, and product category. Export to CSV for accounting.

7.3 Coupons

  • Create discount codes (percent or fixed amount) with optional expiration and scope (memberships, shop, both).
  • Disable a coupon to stop it being redeemed without deleting historical usage.

7.4 QuickBooks Sync

The Admin → QuickBooks page shows the connection status and the last successful sync. Recent invoices mirror over automatically. Reconnect from this page if the OAuth token expires.

13. Maintenance Mode

/admin/site-settings

Maintenance mode locks the platform while the team ships changes. When enabled:

  • An orange "MAINTENANCE MODE ACTIVE" banner shows at the top of every page.
  • Non-admin users are blocked from the site with a "try again later" message.
  • Admins, plus members explicitly granted maintenance bypass via Login Access Control, retain full access.

Before enabling maintenance mode

Confirm any in-flight registrations, voting sessions, or shop orders are tolerable to interrupt. Members mid-checkout will see the maintenance message on their next request.

To toggle:

  1. Open Site Settings.
  2. Flip Maintenance Mode to ON, save.
  3. Refresh any browser tab to see the banner appear site-wide.
  4. When work is complete, flip the toggle back OFF; the banner disappears and full access returns.

14. Login Access Control

/admin/members

The bulk toolbar on the Members page exposes three login-control actions when one or more rows are selected:

14.1 Allow Maintenance Login

Lets a non-admin member sign in while maintenance mode is on (e.g. a beta tester, contractor, or partner needing access during a freeze).

  1. Tick the member rows you want to grant.
  2. Click Allow Maintenance Login.
  3. The row badge updates to "Maint." Click Revoke Maintenance Login later to remove the bypass.

14.2 Ban Login

  1. Tick the rows you want to ban (you cannot ban yourself or a super admin).
  2. Click Ban Login.
  3. Enter a reason of at least 5 characters and confirm.
  4. Active sessions are kicked within ~1 minute and future sign-ins are blocked. The row shows a red Banned badge with the reason in the tooltip.

14.3 Unban Login

The Unban Login button only appears when at least one selected row is banned. Click it to lift the ban; the user can sign in again on their next attempt.

14.4 Audit trail

Every ban / unban / maintenance toggle is recorded in the admin audit log with the admin user, the target, and the reason. Filter audit log by action: login_ban, login_unban, login_allow_maintenance, login_revoke_maintenance.

15. Finals Voting Management

/admin/finals-voting

The Finals Voting page bucketizes sessions into six accordion categories so you always see the right thing at the right time:

BucketWhat lives here
CreatingDrafts being built — no categories yet
PendingDrafts that are configured and ready to open
LiveCurrently open for member voting
DisabledSuspended — hidden from the public regardless of dates
CanceledAborted by an admin — eligible for full deletion
CompletedVoting period ended (closed or finalized)

15.1 Create + run a session

  1. Click + New Session, pick a season, set start/end dates, save.
  2. Add categories and questions (or click Seed 2023 Template).
  3. When ready, click Open Voting; the session moves to Live and the public homepage card appears.
  4. After voting closes, click Finalize & Publish to publish results.

15.2 Suspend a live session

Click the Pause icon on the row (or Suspend in the detail view) to immediately hide the session from the public homepage and prevent any new votes — even mid-voting. Voters' existing ballots are preserved. Click Resume to restore.

15.3 Cancel a session

Use Cancel for sessions you don't intend to run (test sessions, mistakes). The session moves to the Canceled bucket and stays hidden from public surfaces. Cancel is allowed from any non-finalized state.

15.4 Delete a session

Permanent delete is available only for DRAFT or CANCELED sessions. For canceled sessions with cast votes, deletion cascade-clears those votes too — the action is irreversible and double-confirms.

Deleting vs Cancelling

Cancel keeps the session row for audit (just hidden). Delete wipes the entire session and its votes. Prefer Cancel unless you genuinely want no historical record.

17. QA Testing Rounds

/admin/qa-checklist

QA Testing Rounds is the platform's owner-driven sign-off tool. The development team builds the checklist and seeds it; site owners walk through and mark each item Pass / Fail / Skip with optional comments and screenshots; developers see Failed items in a dedicated tab and submit fix notes.

17.1 Create a round with a custom item set

  1. Click + New QA Round.
  2. Title and description.
  3. In Items to include, expand any section and tick/untick individual items, or use Section-level checkboxes / Select all / Clear.
  4. Use the Custom items panel to add ad-hoc items: title, steps (one per line), expected result, optional page URL. Tick Also save to master if it's something every future round should include.
  5. Save. The round opens in DRAFT status with the items you picked.

17.2 Add or remove items mid-round

The Manage Items tab on the round detail page works in DRAFT or Active rounds:

  • + Add from Master — pick more items from the master checklist (already-included items are filtered out).
  • + Add Custom — same composer as on creation, with the same Promote-to-Master toggle.
  • Each item has a remove button (cascades any reviewer responses + developer fix notes; double-confirms when responses exist).
  • Custom items show a star icon to Promote to Master after the fact.

17.3 Suspend, edit, delete a round

From either the round list or the detail page:

  • Edit — change title or description at any point.
  • Suspend — pauses an active round so reviewers cannot submit new responses while developers ship fixes; resume to continue.
  • Delete — permanently removes the round and all responses + fix notes (double-confirms when work exists).

17.4 Manage reviewers

  • Click Add Reviewer on the detail page to assign more admins. They show up on the round list and on their own dashboard.
  • The trash icon on each reviewer row removes them from the round at any status — including Active. Submitted responses for that reviewer are deleted (double-confirms).

17.5 Promote custom items to master

Either tick "Also save to master" when adding the item, or click the Star icon next to a custom item in the Manage Items panel. Promoted items live in qa_master_items and seed into every future round automatically. The badge changes to Custom · Promoted after promotion.
