Add a Judge or Event Director Manually

Four admin paths to certify someone — application approval, full admin entry, quick-add, or direct record

/admin/judge-applications · /admin/event-director-applications · /admin/judges · /admin/event-directors

Two ways to add a Judge or ED

The normal path is application → admin review → approval. But there are admin-only shortcuts for cases where the member can't easily fill out the application themselves (long-time judges grandfathered in, owners adding a friend at an event, etc.). This guide covers all of them.

2. Path A: Approve a member-submitted application

1
Open the queue
Judges: /admin/judge-applications. Event Directors: /admin/event-director-applications. Filter by status = Pending.
2
Open the application
Read the applicant's bio, references, prior judging experience, location, and acknowledgments. Scroll all sections.
3
(Optional) Verify references
Each reference receives a verification email; their response shows on the application page. Wait for at least one reference to confirm before approving for sensitive certifications.
4
Click Approve
Pick the level (e.g. Apprentice, Certified, Senior) and optional certification expiration. Save.
5
What happens next
A Judge (or Event Director) record is created and linked to the user. The user's profile gains can_apply_judge = true (or can_apply_event_director) with grant audit fields populated. They appear in /admin/judges and on the public Judges directory. They can be assigned to event judge panels.

3. Path B: Admin enters a full application on the member's behalf

Useful when a member emails you their info but doesn't navigate the app well. You enter everything as if they had.

1
Open admin application form
From /admin/judge-applications click "+ New Application". Pick the user from the search.
2
Fill every field
Demographics, contact, travel radius, weekend availability, years in industry, specialty, sub-specialties, three references with name/relationship/email/phone/company, the four required acknowledgments, and the three essay sections.
3
Save
The application is created in PENDING state, just like a member-submitted one. entryMethod is set to ADMIN_APPLICATION and enteredBy = your admin profile.
4
Approve
Same approval click-through as Path A.

4. Path C: Admin quick-add (minimal fields)

For when you need a known judge added now, on event day, with the bare minimum.

1
Open quick-add
From /admin/judge-applications click "Quick Add". Pick the user from the search.
2
Provide just the essentials
Full name, phone, country/state/city, travel radius, years in industry, specialty, optional admin notes.
3
Save
The system fills in placeholders for fields the form normally requires (date of birth = 1990-01-01, zip = 00000, weekend availability = BOTH, all four acknowledgments auto-checked, essays auto-filled with "Admin quick entry"). The application is approved immediately and a Judge record is created.

Quick-add bypasses the verification flow

References aren't verified, acknowledgments aren't actually agreed-to by the member, and essays aren't filled. Only use this for judges you personally know and can vouch for. Document the reason in the admin notes field.

5. Path D: Create the Judge / ED record directly

For grandfathering an existing certified judge or ED who never had an application in this system.

1
Open the manage page
Judges: /admin/judges. EDs: /admin/event-directors. Click "+ Direct Add".
2
Pick the user
Search and select the existing member profile.
3
Set certification
Level (e.g. Senior), specialty (SPL / SQL / Show & Shine), grant date, and expiration date.
4
Save
A judges (or event_directors) row is created without an application backing it. The user's profile gets the permission flag set with audit fields. They appear in the public directory and can be assigned to events immediately.

6. Promoting an existing member

If you just want to grant the permission without going through any application, use the Permissions panel on the member detail page:

  • Open /admin/members → member detail.
  • Find the Permissions panel.
  • Toggle "Can apply as Judge" or "Can apply as Event Director" on.
  • Set certification expiration if applicable. Save.

This sets the permission flag with audit fields but does not create a Judge/ED record on its own — they still need to submit an application or you need to use Path D to create the record.

7. Recording a level change after the fact

When a Judge progresses from Apprentice to Certified or up to Senior, log the change so the history is visible.

1
Open the Judge detail
From /admin/judges, click the row.
2
Click "Record Level Change"
Pick the new level, set effective date, optional notes.
3
Save
The current level is updated and a history entry is added so future admins can see the progression.
