Retailer & Manufacturer Account Setup

How retailer / manufacturer accounts and listings work, and how to create them end-to-end

/admin/business-listings · /admin/members

How retailers and manufacturers fit into the platform

A "retailer" or "manufacturer" on MECA is a business that sells car-audio products and wants to be discoverable by competitors. They get a public directory listing, can offer member-only discount codes, can participate as event sponsors, and can advertise via the banner system. Setting one up has both a profile (member account with retailer / manufacturer role) and a listing (the directory entry).

1. The data model — member account vs listing

ConceptWhere it livesWhat it does
Profile with role = retailer or role = manufacturerprofiles tableThe login account. Has email, password, MECA ID. Can log in and manage their own listing.
RetailerListing or ManufacturerListingretailer_listings / manufacturer_listings tablesThe directory entry. Business name, description, logo, address, gallery, special offer text. Linked to the profile via user_id.
Membership with category retail / manufacturermemberships tableOptional — if the retailer wants premium placement / sponsor benefits, they purchase a membership. Without it the listing still works but with reduced perks.

One member can have BOTH a retailer listing and a manufacturer listing if they wear both hats (rare). Most have one or the other.

2. Create a new retailer account from scratch

1
Create the user account
From /admin/members click "+ New Member". Set role = retailer. Account type = Member. Fill name, business email, phone. Password approach: send-reset-link so they set their own.
2
Optionally assign retailer membership
If the retailer is a paying sponsor, assign the appropriate retailer-tier membership and select payment method.
3
Create the listing
Open /admin/business-listings. Click "+ New Retailer Listing". Pick the user account you just created. Fill business name, description, address, store type (brick-and-mortar / online / both), business email, phone, website.
4
Upload images
Profile/logo image (square; ~512×512 minimum). Gallery (up to 6 images, optional product link per image). Cover image position adjusted by drag.
5
Set offer text (optional)
A short pitch for MECA members — discount code, exclusive deal, anything that runs on the public retailer profile.
6
Approve / activate
Toggle isActive and isApproved on. Save. The listing appears on the public Retailers directory and on the user's "My Business" page when they log in.

3. Create a new manufacturer account from scratch

Same flow as retailer, but role = manufacturer, listing goes into /admin/business-listings → Manufacturers tab, and there's no storeType field. The manufacturer listing has additional fields like product categories and supported formats. Otherwise the click-through is identical.

4. Approving a self-submitted listing

If a retailer / manufacturer signed up themselves and filled their listing, it lands in /admin/business-listings with isApproved = false. To approve:

1
Filter pending
Open /admin/business-listings and filter by approval status = Pending.
2
Verify the business
Open the listing — confirm the business actually exists (Google their website, check Maps for the address, verify the phone). Make sure the description is appropriate.
3
Approve or reject
Approve flips isApproved = true and the listing goes live in the public directory. Reject with a reason — the user is notified.

6. Sponsor settings & member discount codes

  • Sponsor flag — flip isSponsor = true to give the listing priority placement in the directory and a slot on the homepage carousel.
  • Member discount codes — the offer text field is the simple way; for actual coupon-code redemption at the MECA Shop, create a coupon at /admin/coupons and tell the retailer to advertise it via their offer text.
  • Banner ads — sponsors typically also run banner ads. See the Banner Ads guide.

7. Deactivate or remove a listing

  • Deactivate — toggle isActive = false. Removes from the public directory but the data is kept. Use for businesses that close temporarily.
  • Delete — use only for clearly-named test entries or duplicates. Cascades any references and cannot be undone.
  • Suspend the user account — if the retailer is misbehaving (spamming, fraudulent offers), ban the underlying user account; the listing automatically becomes hidden because the underlying user can no longer log in to manage it.
